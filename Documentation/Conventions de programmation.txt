--- CONVENTIONS DE PROGRAMMATION DU SYSTEME ---
--- V 1.0, 16/11/11                         ---
-----------------------------------------------

- Nommage des fonctions :
-------------------------

Le nom des fonctions est constitué de 3 parties, chacune commençant par une majuscule (style Pascal) : SectionActionDonnée
ex : 
# fonction écrivant une chaîne dans la console : ConsoleWriteString()
# fonction envoyant un octet via le port série (appelé aussi Universal Asynchronous Receiver Transmitter) : UARTWriteByte()
# fonction renvoyant la dernière touche du clavier qui a été pressée : KeyboardReadKey()


- Nommage des variables :
-------------------------

Le nom des variables est composé d'un ou plusieurs mots commençant tous par des majuscules, les mots sont séparés par le caractère underscore (_) : Test_Variable
ex : Compteur, Nombre_Elements, Chaine_Tampon


- Nommage des types :
---------------------

Le nom des types est constitué d'un ou plusieurs mots, séparés entre eux par une Majuscule. Le type commence par un T majuscule (style Pascal) : TTypeDonnee
ex : TFiche, TChaineUnicode


- Nommage des constantes :
--------------------------

Les constantes sont composées d'un ou plusieurs mots en majuscules, séparés par des underscores (_) : TAILLE_TABLEAU
ex : NOMBRE_ENTREES_TABLEAU, PI

- Ordre des "sections" dans un fichier source :
-----------------------------------------------

On désigne par "section" les parties de code qui déclarent des variables, les macros et les implémentations de fonctions.
Par soucis d'uniformité, voici l'ordre d'apparition de ces sections dans un fichier source qui a été retenu :
# constantes (#define)
# macros (#define)
# variables "privées" (static)
# variables "publiques" (extern dans le .h)
# fonctions privées (static)
# fonctions publiques (leur prototype se trouve dans le .h)
